Домашнее задание: Расчет входящего урона по мобу ⚔️
Описание Задачи
Представьте, что вы разрабатываете свою игру. Один из ключевых элементов — боевая система. Вам нужно написать программу-калькулятор, которая будет рассчитывать, сколько урона получит монстр (моб) после атаки игрока, учитывая его броню.

Основная функциональность

В вашей программе у моба есть два основных параметра: здоровье (HP) и броня (в процентах). Задайте эти значения прямо в коде. Например, "Гоблин" со 100 HP и 20% брони.
Программа должна запрашивать у пользователя базовый урон его атаки.
После ввода, программа рассчитывает итоговый урон по формуле. Броня снижает входящий урон на свой процент.
В конце программа должна вывести подробный результат: сколько урона заблокировала броня, какой чистый урон был нанесён и сколько здоровья осталось у моба.
Дополнительные условия / Ограничения

Используйте только стандартные средства Java.
Параметры моба (изначальное здоровье, процент брони) должны быть заданы как константы (final).
Примеры использования

Пример сценария:
Вы запускаете программу. В коде задан Гоблин (100 HP, 20% брони).
Программа выводит: Введите базовый урон вашей атаки:
Пользователь вводит 50.
Программа выполняет расчеты и выводит результат.
Входные / Выходные данные
Вход:

Одно число (целое или дробное) — базовый урон атаки.
Выход:

Подробный текстовый отчёт в консоли.
Атака наносит 50.0 базового урона...
Броня гоблина (20%) заблокировала 10.0 урона.
Итоговый урон по цели: 40.0.
Здоровье гоблина: 60.0 / 100.0.
Технические требования
Среда выполнения: Консоль.
Бонусные Задания (Необязательно)
Критические удары: Добавьте в расчёт шанс критического удара. Например, с вероятностью 25% атака наносит двойной урон (до расчёта брони). Сообщайте пользователю, если удар был критическим.

Пример вывода: Критический удар! Базовый урон увеличен с 10.0 до 100.0.
Классы и Объекты: Создайте класс Mob, у которого будут поля name, health, maxHealth, armorPercent. В классе создайте метод takeDamage(double baseDamage), который будет выполнять все расчеты и изменять здоровье объекта. В main методе создайте экземпляр этого класса и вызовите его метод.

Битва до победы: Организуйте бой в цикле (while). После каждой атаки выводите оставшееся здоровье моба и предлагайте нанести следующий удар. Цикл должен продолжаться, пока здоровье моба не упадёт до 0 или ниже. В конце выведите победное сообщение.

Добавьте магию в этот скучный мир: Сделайте так, чтобы у урона появилась "стихия". А у мобов - сопротивление к ней
