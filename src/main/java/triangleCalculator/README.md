# Домашнее задание (сложное): Расчет параметров прямоугольного треугольника ## **Описание Задачи** В этом домашнем задании вам предстоит создать комплексную программу на Java, которая будет запрашивать у пользователя два известных параметра прямоугольного треугольника и на их основе вычислять все остальные неизвестные стороны, углы, а также площадь и периметр. Это задание поможет вам углубить знания по математическим операциям, условным конструкциям (`if-else` или `switch`), использованию функций класса `Math` и обработке ввода пользователя. * **Основная функциональность** * Программа должна предложить пользователю выбрать, какие два параметра прямоугольного треугольника ему известны: * Два катета (a, b) * Катет и гипотенуза (a, c) или (b, c) * Катет и острый угол (a, alpha) или (a, beta) * Гипотенуза и острый угол (c, alpha) или (c, beta) * После выбора программа должна запросить значения этих двух известных параметров. * На основе введенных данных, программа должна вычислить все **неизвестные** параметры треугольника: * Длины всех сторон (a, b, c) * Величины всех углов (alpha, beta, 90) * Площадь (S) * Периметр (P) * Все вычисленные значения должны быть выведены на экран с точностью до двух знаков после запятой. * **Обработка некорректного ввода и невозможных треугольников:** * Программа должна проверять корректность введенных значений (например, стороны не могут быть отрицательными или нулевыми; гипотенуза не может быть короче катета; острые углы должны быть от 0 до 90 градусов). * В случае некорректных данных или невозможности построить треугольник с заданными параметрами, программа должна вывести соответствующее сообщение об ошибке и завершить работу. * **Дополнительные условия / Ограничения** * Используйте только стандартные библиотеки Java. * Все данные должны храниться в памяти. * Для вычислений используйте `double`. * Помните, что тригонометрические функции в Java (`Math.sin`, `Math.cos`, `Math.tan`) работают с **радианами**, поэтому необходимо правильно конвертировать градусы в радианы и обратно, используя `Math.toRadians()` и `Math.toDegrees()`. * Используйте `if-else if` или `switch` для выбора логики расчета в зависимости от того, какие параметры известны. * **Примеры использования** * **Пример сценария 1 (Известны два катета):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Какие параметры известны? (1-Катеты, 2-Катет и гипотенуза, 3-Катет и угол, 4-Гипотенуза и угол):" 3. Пользователь вводит `1`. 4. Приложение запрашивает: "Введите длину первого катета (a):" 5. Пользователь вводит `3.0`. 6. Приложение запрашивает: "Введите длину второго катета (b):" 7. Пользователь вводит `4.0`. 8. Приложение выводит: ``` Расчеты для прямоугольного треугольника: Катет a: 3.00 Катет b: 4.00 Гипотенуза c: 5.00 Угол альфа (напротив a): 36.87 градусов Угол бета (напротив b): 53.13 градусов Площадь (S): 6.00 Периметр (P): 12.00 ``` * **Пример сценария 2 (Некорректный ввод - отрицательная сторона):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Какие параметры известны?..." 3. Пользователь вводит `1`. 4. Приложение запрашивает: "Введите длину первого катета (a):" 5. Пользователь вводит `-5.0`. 6. Приложение выводит: ``` Ошибка: Длины сторон не могут быть отрицательными или нулевыми. ``` 7. Приложение завершает работу. * **Пример сценария 3 (Невозможный треугольник - катет длиннее гипотенузы):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Какие параметры известны?..." 3. Пользователь вводит `2`. 4. Приложение запрашивает: "Введите длину катета (например, a):" 5. Пользователь вводит `5.0`. 6. Приложение запрашивает: "Введите длину гипотенузы (c):" 7. Пользователь вводит `4.0`. 8. Приложение выводит: ``` Ошибка: Гипотенуза не может быть короче катета. Такой треугольник не существует. ``` 9. Приложение завершает работу. ----- ## Входные / Выходные данные **Входные данные:** * **Выбор опции:** Целое число (1-4), указывающее тип известных параметров. * **Значения параметров:** Два числа с плавающей точкой (катеты, гипотенуза, углы в градусах). **Выходные данные:** * Подробный отчет по всем вычисленным параметрам (стороны, углы, площадь, периметр) с точностью до двух знаков после запятой. * Сообщения об ошибках при некорректном вводе или невозможности построить треугольник. ----- ## Технические требования * Используйте `java.util.Scanner` для считывания ввода пользователя. * Активно применяйте математические функции из класса `java.lang.Math` (`Math.sqrt()`, `Math.pow()`, `Math.sin()`, `Math.cos()`, `Math.tan()`, `Math.asin()`, `Math.acos()`, `Math.atan()`). * Используйте `if-else if` или `switch` для разветвления логики программы в зависимости от выбранного типа известных параметров. * Примените `String.format("%.2f", value)` для форматирования вывода чисел с плавающей точкой. * **Среда выполнения:** Консоль ----- ## **Бонусные Задания (Необязательно)** * **Циклический режим работы:** * Позволить программе выполнять расчеты многократно, пока пользователь не введет команду для выхода (например, " exit"). * **Проверка суммы углов:** * После всех расчетов, убедитесь, что сумма всех трех углов треугольника (два острых и прямой угол) приблизительно равна 180 градусам (с учетом возможных погрешностей вычислений с плавающей точкой, например, 0.01 градуса). Если нет, вывести предупреждение. * **Бонусные баллы:** +20% к итоговой оценке за каждое выполненное бонусное задание.
