# Домашнее задание: Проверка Палиндрома (с использованием StringBuilder и toCharArray) ## **Описание Задачи** В этом домашнем задании вам предстоит написать программу, которая определяет, является ли введенная пользователем строка **палиндромом**. Для реализации этой проверки вам необходимо использовать классы `StringBuilder` и метод `String.toCharArray()`. **Палиндром** — это слово, фраза, число или другая последовательность символов, которая читается одинаково как с начала, так и с конца (например, "довод", "казак", "мадам"). * **Основная функциональность** * Программа должна запросить у пользователя **одну строку**. * Реализуйте метод `public static boolean isPalindrome(String text)`: * Этот метод должен принимать строку в качестве аргумента. * **Внутри метода:** 1. Преобразуйте входную строку в массив символов (`char[]`) с помощью метода `toCharArray()`. 2. Создайте новый `StringBuilder`. 3. Используя массив символов, заполните `StringBuilder` символами **в обратном порядке**. 4. Преобразуйте `StringBuilder` обратно в строку. 5. Сравните исходную строку (после приведения к нижнему регистру и удаления пробелов/знаков препинания) с полученной перевернутой строкой (также после приведения к нижнему регистру и удаления пробелов/знаков препинания). 6. Верните `true`, если строки совпадают, иначе `false`. * Выведите результат проверки на экран, сообщая пользователю, является ли его строка палиндромом. * **Дополнительные условия / Ограничения** * **Игнорирование регистра:** При сравнении строк игнорируйте регистр букв (например, "ДоВоД" должен быть палиндромом). Используйте `String.toLowerCase()`. * **Игнорирование пробелов и знаков препинания:** При проверке палиндрома должны игнорироваться пробелы, знаки препинания и любые другие символы, не являющиеся буквами. Например, "А роза упала на лапу Азора" - это палиндром. Для этого можно использовать метод `Character.isLetter()` и фильтровать только буквы перед формированием перевернутой строки. * * **Примеры использования** * **Пример сценария 1 (простой палиндром):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите строку для проверки на палиндром:" 3. Пользователь вводит `довод`. 4. Приложение выводит: ``` Строка "довод" является палиндромом. ``` 5. Приложение завершает работу. * **Пример сценария 2 (палиндром с разными регистрами и пробелами):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите строку для проверки на палиндром:" 3. Пользователь вводит `А роза упала на лапу Азора`. 4. Приложение выводит: ``` Строка "А роза упала на лапу Азора" является палиндромом. ``` 5. Приложение завершает работу. * **Пример сценария 3 (не палиндром):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите строку для проверки на палиндром:" 3. Пользователь вводит `привет`. 4. Приложение выводит: ``` Строка "привет" не является палиндромом. ``` 5. Приложение завершает работу. * **Пример сценария 4 (пустая строка):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите строку для проверки на палиндром:" 3. Пользователь вводит (нажимает Enter, не вводя ничего). 4. Приложение выводит: ``` Строка не может быть пустой. Пожалуйста, введите что-то. ``` 5. Приложение снова запрашивает ввод. (Опционально, можно просто завершить работу или обработать пустую строку как не-палиндром, но лучше запросить повторный ввод). ----- ## Входные / Выходные данные **Входные данные:** * Одна строка текста, введенная пользователем. **Выходные данные:** * Сообщение о том, является ли введенная строка палиндромом. * Сообщения об ошибках при некорректном/пустом вводе. ----- ## Технические требования * **Среда выполнения:** Консоль ----- ## **Бонусные Задания (Необязательно)** * **Рекурсивная проверка палиндрома:** * Реализуйте второй метод `public static boolean isPalindromeRecursive(String text)` для проверки палиндрома * *рекурсивно**, без использования `StringBuilder` или `toCharArray` (работайте напрямую со строкой и индексами).
