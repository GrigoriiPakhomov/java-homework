# Домашнее задание: Проблема FizzBuzz

## Описание задачи

Это домашнее задание поможет вам закрепить навыки работы с **циклами (`for` или `while`)**, **условными операторами (`if-else if-else`)** и **оператором остатка от деления (`%`)**.

Вам предстоит решить классическую "проблему FizzBuzz", которая часто используется для проверки базовых навыков программирования.

---

## Основная функциональность

- Программа должна запросить у пользователя **целое число `N`**, которое будет верхней границей диапазона.
- Затем программа должна пройти по всем числам от `1` до `N` (включительно).
- Для каждого числа выполните следующие проверки и выведите результат:
    - Если число делится на `3` и на `5` без остатка (т.е. на `15`), выведите `"FizzBuzz"`.
    - Иначе, если число делится на `3` без остатка, выведите `"Fizz"`.
    - Иначе, если число делится на `5` без остатка, выведите `"Buzz"`.
    - Во всех остальных случаях выведите само число.

---

## Дополнительные условия / Ограничения

- Используйте только стандартные библиотеки Java.
- Реализуйте обработку **некорректного ввода**:
    - Если пользователь введет нечисловое значение, программа должна вывести сообщение об ошибке (например, `"Ошибка: Введите целое число."`) и корректно обработать это исключение (`InputMismatchException` или `NumberFormatException`) без завершения программы, позволяя пользователю ввести число заново.
    - Если пользователь введет отрицательное число или ноль, выведите сообщение об ошибке (например, `"N должно быть положительным числом."`) и попросите ввести число заново.

## Входные / Выходные данные

### Входные данные:

- Одно целое число `N`.

### Выходные данные:

- Последовательность чисел или слов (`"Fizz"`, `"Buzz"`, `"FizzBuzz"`), каждое на новой строке.
- Сообщения об ошибках при некорректном или невалидном вводе.

## Бонусные задания (необязательно)

### Настройка делителей

- Позвольте пользователю вводить не только `N`, но и два других числа, которые будут использоваться вместо `3` и `5` для проверки делимости.

### Сохранение в строку

- Вместо немедленного вывода каждого элемента, сохраняйте всю последовательность в одну строку, используя `StringBuilder` или `StringBuffer`, а затем выведите её полностью в конце.
- Разделяйте элементы пробелом или запятой.
