# Домашнее задание: Проверка даты на валидность ## **Описание Задачи** Это домашнее задание поможет вам отработать использование условных операторов (`if-else`), логических операций и обработку ввода данных в Java. Вы разработаете программу, которая будет запрашивать у пользователя день, месяц и год, а затем определять, является ли введенная дата корректной. * **Основная функциональность** * Программа должна запросить у пользователя **день** (целое число). * Программа должна запросить у пользователя **месяц** (целое число). * Программа должна запросить у пользователя **год** (целое число). * После ввода всех трех значений, программа должна проверить их на валидность согласно следующим правилам: * **Месяц** должен быть в диапазоне от 1 до 12. * **День** должен быть в диапазоне от 1 до 31. * **Количество дней в месяце:** * Апрель (4), Июнь (6), Сентябрь (9), Ноябрь (11) имеют 30 дней. * Январь (1), Март (3), Май (5), Июль (7), Август (8), Октябрь (10), Декабрь (12) имеют 31 день. * Февраль (2) имеет 29 дней в високосном году и 28 дней в невисокосном году. * **Високосный год:** Год является високосным, если он делится на 4, но не делится на 100, ИЛИ если он делится на 400. * Примеры високосных годов: 2000, 2004, 2024. * Примеры невисокосных годов: 1900, 2100, 2023. * Выведите сообщение о том, является ли дата валидной, и если нет, то почему. * **Примеры использования** * **Пример сценария 1 (Валидная дата):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите день:" 3. Пользователь вводит `15`. 4. Приложение запрашивает: "Введите месяц:" 5. Пользователь вводит `3`. 6. Приложение запрашивает: "Введите год:" 7. Пользователь вводит `2023`. 8. Приложение выводит: ``` Дата 15.3.2023 является валидной. ``` 9. Приложение завершает работу. * **Пример сценария 2 (Некорректный месяц):** 1. Пользователь запускает приложение. 2. Приложение запрашивает: "Введите день:" 3. Пользователь вводит `1`. 4. Приложение запрашивает: "Введите месяц:" 5. Пользователь вводит `13`. 6. Приложение запрашивает: "Введите год:" 7. Пользователь вводит `2024`. 8. Приложение выводит: ``` Дата 1.13.2024 невалидна: Месяц должен быть от 1 до 12. ``` 9. Приложение завершает работу. ----- ## Входные / Выходные данные **Входные данные:** * Три целых числа: **день**, **месяц** и **год**. **Выходные данные:** * Строка, указывающая, является ли дата валидной, и, в случае невалидности, причину. * Сообщение об ошибке, если ввод некорректен. ----- ## Технические требования * **Среда выполнения:** Консоль ----- ## **Бонусные Задания (Необязательно)** * **Определить день недели:** * После проверки на валидность, если дата корректна, рассчитайте и выведите день недели для этой даты (например, " Вторник"). Для этого можно использовать алгоритм Зеллера или подобные. (Это может быть сложно для уровня, если они не знают циклы и массивы, но стоит упомянуть как вызов).
