# Задание: Консольная битва покемонов

## **Описание задачи**

Твоя задача — создать консольную симулятор-игру **"Битва покемонов 1х1"**. Игроки будут управлять своими покемонами, выбирая атаки или предметы, чтобы победить покемона противника.

---

## **Основная функциональность**

### **Класс `Pokemon`**

Главный класс, представляющий покемона. У него должны быть следующие поля:

* `name` (имя покемона)
* `hp` (текущие очки здоровья)
* `attack` (сила обычной атаки)
* `level` (уровень покемона)

### **Игровая логика**

* Два покемона (один игрока, один компьютера) начинают битву с полным здоровьем
* Очередность ходов: каждый ход игроки по очереди выбирают действие

### **Действия**

**Обычная атака**

Покемон наносит урон, равный его силе атаки, покемону противника

**Использование предмета**

Игрок может использовать предмет, чтобы восстановить здоровье своего покемона. Предметы должны храниться в инвентаре

### **Условия победы / поражения**

* Битва заканчивается, когда здоровье одного из покемонов опускается до нуля или ниже
* Победитель получает опыт и повышает свой уровень
* Увеличение уровня должно повышать характеристики покемона (`hp`, `attack`)

### **Классы для атак и предметов**

Создай абстрактный класс `Skill` или `Move` с методом:

```
use(Pokemon target)
```

Наследуй от него:

* `Attack` (наносит урон)
* `Heal` (восстанавливает здоровье)

Это позволит в будущем легко добавлять новые типы атак и заклинаний

---

## **Технические требования**

### **ООП**

Используй:

* наследование
* инкапсуляцию
* полиморфизм
* при необходимости интерфейсы

для создания гибкой архитектуры

### **Ввод / вывод**

Взаимодействие с игроком должно осуществляться через консоль

### **Математика**

Расчет урона, опыта и роста характеристик должны быть реализованы логически

---

## **Бонусные задания (необязательно)**

### **Специальные атаки**

Добавь в игру способности, которые:

* наносят больше урона
* имеют дополнительные эффекты (например, «оглушить» противника)

### **Типы покемонов**

Введи систему стихий:

* Огонь
* Вода
* Трава

и зависимость силы атаки от типа
(например, Огонь наносит двойной урон Траве)

---

## **Follow-up: Командные бои (сложное)**

После успешной реализации и сдачи базовой версии, ты можешь приступить к расширению игры до командных битв

### 1. Командные бои

Измени логику игры, чтобы она поддерживала битвы:

* 2×2
* 3×3
* 4×4

### 2. Массовые атаки

Реализуй атаки, которые могут наносить урон нескольким противникам одновременно

Это потребует изменения методов, принимающих не одного, а несколько покемонов

### 3. ИИ противника

Разработай простой искусственный интеллект для команды противника, который будет принимать решения на основе текущего состояния игры

Например:

* атаковать самого слабого покемона
* использовать лечение, когда здоровье низкое

### 4. Управление командой

Добавь возможность игроку переключаться между покемонами в своей команде

Эти дополнения превратят твой проект из простой консольной игры в полноценный симулятор, демонстрирующий твое мастерство в проектировании сложных систем
